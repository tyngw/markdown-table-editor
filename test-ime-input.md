# 日本語IME入力テスト

このファイルは、日本語入力時のIME（Input Method Editor）と編集確定Enterキーの動作を区別するテストです。

## テーブル

| 日本語 | English | 数字 |
|--------|---------|------|
| こんにちは | Hello | 123 |
| さようなら | Goodbye | 456 |
| ありがとう | Thank you | 789 |

## IME入力テスト手順

### 基本的な日本語入力テスト

1. VSCodeでこのファイルを開く
2. `Ctrl+Shift+P` → "Markdown Table Editor: Edit Table" を実行
3. 任意のセルを編集モードにする
4. **日本語入力モード**に切り替える（通常は`半角/全角`キー）
5. 「にほんご」と入力する
6. **Enterキー**を押して漢字変換を確定する → 「日本語」に変換される
7. 編集が継続されることを確認する（セルから出ない）
8. **再度Enterキー**を押す → 編集確定し、下の行に移動することを確認

### 複雑な変換のテスト

1. 新しいセルで「きょうはいいてんきです」と入力
2. **Enterキー**で変換候補を確定 → 「今日はいい天気です」
3. 編集が継続されることを確認
4. **Shift+Enter**で改行を追加
5. 「あしたもはれるでしょう」と入力
6. **Enterキー**で変換確定 → 「明日も晴れるでしょう」
7. **Enterキー**で編集確定し、下の行に移動

### 英数字混在テスト

1. 新しいセルで「VSCode2024ねん」と入力
2. **Enterキー**で変換 → 「VSCode2024年」
3. 編集継続を確認
4. **Ctrl+Enter**で編集確定（移動なし）

## 期待される動作

### IME入力中のEnter
- **日本語変換中のEnter**: 変換確定のみ（編集継続）
- **変換確定後のEnter**: 編集確定 + 下の行に移動

### その他のキー
- **Shift+Enter**: 改行入力（編集継続）
- **Ctrl+Enter**: 編集確定のみ（移動なし）
- **Tab**: 編集確定 + 次のセルに移動
- **Escape**: 編集確定のみ（移動なし）

## 技術的な検証ポイント

1. `compositionstart`イベントで`isComposing = true`になる
2. `compositionend`イベントで`isComposing = false`になる
3. `isComposing = true`の間はEnterキーで編集確定されない
4. `isComposing = false`の時はEnterキーで編集確定される
5. 開発者ツールのコンソールでIME状態のログが確認できる
