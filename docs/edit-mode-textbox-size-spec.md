# 編集モードテキストボックスサイズ仕様

## 概要
React版テーブルエディターにおける編集モード中のテキストボックスサイズに関する仕様です。

## 期待される動作

### 基本原則
- セルの編集を開始した際に表示されるテキストボックスは、その行の全セルのうち最も高さが大きいセルに合わせてサイズを設定する
- テキストボックスの高さは、視覚的にその行の他のセルと統一感を保つ

### 具体的な要件

#### サイズ計算
1. 編集対象セルが属する行の全セルの高さを測定
2. その中から最大の高さを取得
3. テキストボックスの高さを最大高さに設定

#### 対象となるセル内容
- 短いテキスト（1行）
- 長いテキスト（複数行に折り返される場合）
- 改行を含むテキスト
- HTMLエンティティを含むテキスト
- 特殊文字を含むテキスト
- 空のセル

#### 期待されるケース
- **複数行テキストがある行**: 最も高いセルに合わせてテキストボックスが表示される
- **異なる文字数のセル**: 表示に必要な高さが最も大きいセルに合わせる
- **改行を含むセル**: 改行による高さ増加も考慮される
- **空セル混在**: 空セルがあっても、他のセルの最大高さに合わせる

#### 想定される問題ケース
以下のようなケースで正しく動作しない可能性がある：
- セルに特殊な文字エンコーディングが含まれる場合
- CSSによる表示高さと実際の内容高さが異なる場合
- フォントサイズやline-heightが影響する場合
- HTMLエスケープが影響する場合

## 品質要件
- すべてのセル内容タイプで一貫した動作を示す
- ユーザーの編集体験に違和感がない
- パフォーマンスに大きな影響を与えない
